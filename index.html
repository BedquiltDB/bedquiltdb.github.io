BedquiltDB

A Json document store on PostgreSQL

CREATE EXTENSION bedquilt;

What?

BedquiltDB is an open-source PostgreSQL extension and set of code libraries, which wraps the PostgreSQL jsonb data type in a slick programmatic API inspired by NoSQL document stores.

Why?

Many developers prefer the code-centric APIs of NoSQL stores like MongoDB, especially in the early stages of a project, but would like to still have all the power of an SQL database available without going through a painful migration.

BedquiltDB aims to provide developers with a powerful tool, both in early stages of a project, and when explicitly dealing with loosely-structured data, while providing an easy migration path into SQL-land once the schemas fall into place.

Advantages

  • Simple, developer-friendly programmatic API
  • Easy to write clients in any programming language, as all the "smarts" are in the PostgreSQL extension
  • All the power of a SQL database, including Constraints and Transactions
  • Easy migration path from "loose schema" to "well defined schema" without needing to change database platform
  • If there's something you want that BedquiltDB doesn't support directly, you can just use ordinary SQL, BedquiltDB won't stop you!

Examples

Python

import pybedquilt
bq = pybedquilt.BedquiltClient(dbname='bedquilt_test')
projects = bq['projects']

projects.add_constraints({
    'description': {'$required': 1,
                    '$notnull': 1,
                    '$type': 'string'},
    'tags': {'$required': 1,
             '$type': 'array'}
})

projects.insert({
    '_id':   "BedquiltDB",
    'description': "A ghastly hack.",
    'quality': "pre-alpha",
    'tags': ["json", "postgres", "api"]
})

early_projects = projects.find(
    {'quality': 'pre-alpha'})

api_projects = projects.find(
    {'tags': ['api']})

cool_project = projects.find_one_by_id('BedquiltDB')

Node.js

var BedquiltClient = require('bedquilt').BedquiltClient;

BedquiltClient.connect('localhost', function(err, client) {
  var tools = client.collection('tools');
  tools.find({type: 'blunt'}, function(err, results) {
    console.log(results);
  });
});

SQL

select bq_insert(
  'orders',
  '{"_id": "2001515",
    "customerId": "117176",
    "items": ["orange_coat", "socks", "plastic_teeth"]}'
);

select bq_find(
  'orders',
  '{"customerId": "220011",
    "items": ["socks"]}'
);

select bq_remove_one(
  'orders',
  '{"_id": "4401515"}'
);

Features

Core features:

  • Easy-to-install PostgreSQL extension
  • Automatically create collections on first write
  • Insert and Save data to a collection
  • Automatic generation of id fields for inserted documents
  • Add schema constraints to collections
  • Find data documents from collections
  • Remove data from collections


Client (Driver) Libraries:

  • Python (alpha)
  • Node.js (alpha)

Roadmap

Planned Features:

  • Add indexes to collection fields
  • Advanced query operations
  • Update, with partial modification of documents
  • Client control of Transactions
  • Collection namespaces


Planned Client Libraries:

  • Python
  • Clojure
  • Javascript
  • Ruby
  • Scala
  • Go
  • And many more...

Projects

  • bedquilt-core - The main PostgreSQL extension. (docs)
  • pybedquilt - The official Python driver for BedquiltDB. (docs)
  • node-bedquilt - The official Node.js driver for BedquiltDB. (docs)
